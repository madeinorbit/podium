The Closed Fold — quiet without erasing history

Tillmann types “close and archive” because closing the work does not quiet its row. That second command is presentation debt. Closed top-level issues should settle into one restrained disclosure; close becomes enough, while archive keeps its precise meaning.

Recommended policy

Auto-fold, never auto-archive. Folding changes the sidebar projection only. Archiving remains an explicit record-lifecycle action for the rare moment someone actually wants removal from every list.

visual decay · no data decay
01 · The four laws

Closed exits the conversation, not the record

F1

Closed folds; done does not

Only closed top-level issues enter the strip. POD-100 decision 15 remains intact for done children and their shipped decay.

F2

Unread resists folding

A closed issue with an unseen result keeps its full row and plain blue beacon. Age never overrides attention.

F3

Selection finishes the handoff

Opening the result marks it read, but the selected row stays put. It folds only after focus moves away, so the interface never disappears under the pointer.

F4

Archive stays explicit

The strip has no timer and performs no background mutation. Manual archive still means immediate removal everywhere; routine sidebar cleanup no longer needs it.

02 · Live state model

One quiet line carries four settled issues

PlacementThe fold sits at each project group’s tail, preserving provenance and group arithmetic. A global graveyard would make the user re-discover which project each issue belonged to.
Visual grammarSuperade navy, tint-never-fill rows, static unread blue, and rail-navy-adjacent seams come directly from POD-100 final rev 2. The fold adds no attention color and no ambient motion.

Expand/collapse is local UI state. Expanded rows remain navigable and manually archivable; collapsing them changes neither order nor identity.

04 · Implementation boundary

A projection rule after POD-100 lands

Fold membershiptop-level ∧ closed ∧ read ∧ not-selected. Closed unread and the currently selected closed issue remain full rows above the strip.
ArithmeticThe project-group count includes every issue exactly once. Expanding the strip changes rendered rows, never totals. Closed · N counts only rows currently inside the fold.
OrderFolded rows retain their POD-168 manual sortKey; expansion uses that stable order. Folding never rewrites order.
MutationNone. No implicit archive timestamp, grace-period worker, ownership rewrite, or session decay is introduced.
DependencyImplementation waits for POD-100’s sidebar rollout, then lands as derived grouping in derive.ts plus a disclosure in SidebarUnified.tsx.
