02 · The design

Pinned above everything, groups below, nothing ever moves on its own

★Pinned
Podium

Try it

  1. New task — mints a row at the top of its group: it expands into place, everything below slides down, an issue-tint wash decays. One shot, then stillness (R2).
  2. Drag the ⠿ grip (hover a row's left edge) to reorder. Order is yours; nothing reshuffles it (R1).
  3. Click rows — selection is a surface: 28% tint, inset ring, and the color band crossing this seam (R6, R9). The unread dot is a beacon; it never looks like selection.
  4. Pinned lives above all groups — its rows keep full identity and state, Linear-favorites style (R3).
  5. The working row runs the braille spinner + timer in Superade live-red — the only thing moving on this panel (R5).
03 · Ordering model

Manual order needs one new field; everything else already exists

The server has no order field today — IssueWire carries priority, pinned, seq, createdAt, deferUntil and nothing positional (protocol/messages/issues.ts). The client's fixed sort is pinned-band → creation-desc (derive.ts:1600).

RuleDefinition
New field: sortKeyFractional/lexo-rank string on the issue (Linear's approach): reorder writes one row, no renumbering. Server: issues module + issues.update patch.
Default key = top of groupA new issue mints a key above the group's current minimum — R2's “new appears at top” is the sort's natural behavior, not a special case.
Band model (unchanged shape)PINNED section (own manual order, above all groups) → project groups (manual order) → snoozed sink to the group tail exactly as today (derive.ts:1569).
Pinning = move, not copyA pinned row leaves its project group for the PINNED section (no duplicates in a 272px column); unpinning returns it to its sortKey position. Group label shows on the pinned row's tooltip.
Drag affordanceA 10px grip zone (⠿) fades in at the row's left edge on hover; drag lifts the row (tight transient shadow — allowed, it disappears), siblings FLIP 180ms. Cross-group drops re-home only within the same project; into/out of PINNED toggles pinned.
What never sortsUrgency, activity, updatedAt — #64 stays law. Attention renders in the row (yellow pill, blue dot), never as motion of the row.
04 · Arrival motion

“Arrive” — a new one-shot in the existing grammar

The motion system has morphs for ignite, flash, pop, flip — but nothing for row insertion. Arrive joins them, obeying the same law: one shot, ease-out, then total stillness.

PhaseSpec
1 · Make roomThe new row's wrapper expands height: 0 → auto over 260ms cubic-bezier(.22,1,.36,1). Every row below slides down as a consequence — no per-row transforms, no layout thrash beyond one element.
2 · MaterializeRow content fades in and settles 4px, 300ms, delayed 60ms so the gap visibly opens first — the eye reads “space was made, then filled”.
3 · WashAn issue-tint wash (26% mix) decays to transparent over ~1s — the row marks itself as the newcomer without any badge. Yellow is not used: arrival is information, not a request for you.
Reduced motionRow appears instantly; no expansion, no settle; spinner frames freeze. Matches the existing motion.css:184 gate.
05 · State grammar

Selection is a surface; unread is a beacon

They share zero visual properties, so they can never be confused — and both survive composition with tint, working, and needs-you states.

Selectedsurface · tint 28% + ring + notch + strong ink
Unreadbeacon · blue dot + medium weight, surface untouched
Workingred spinner + timer · the only motion
Needs youyellow pill + frozen ago-stamp · zero motion

Two deliberate rules: the unread dot hides while its row is selected (you are looking at it), and selection's weight bump goes to 600 while unread's stays 500 — weight alone never distinguishes them, the surface does. Superade note: the shipped code still times work in green #10b981; DESIGN.md's Superade grammar maps live activity to red and has no green — the mock follows DESIGN.md.
