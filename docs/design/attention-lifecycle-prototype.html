Design laws 5
01

Attention outranks read

Reading clears the blue dot. It never clears an offer, question, or review obligation.

02

Issue owns visibility

Planning, working, and review issues stay addressable even with no active agent.

03

Sessions are execution

Pause frees compute. Retire hides a conversation. Neither closes the issue.

04

Issue views stay scoped

Only task-owned changes or delivery state appear here. Checkout-wide activity belongs in Git and workspace views.

05

Closed is chronological

Newest closures sit first. Hover Archive removes a quiet row from the sidebar; manual sorting ends at close.
