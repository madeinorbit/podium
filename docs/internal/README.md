# Internal working notes

Everything under `docs/internal/` is internal working material — implementation plans, session handoffs, investigation notes, historical migration reports, and design scratch — kept in the repository for provenance and archaeology. These files are **not product documentation**: they may be outdated, describe abandoned directions, or reference development-time setups, and they are not part of releases. For user- and contributor-facing documentation, see the rest of `docs/` and the root `README.md` / `ARCHITECTURE.md` / `CONTRIBUTING.md`.
