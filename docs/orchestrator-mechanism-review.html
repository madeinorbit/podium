What the adversarial reviews caught

Every slice went implement → adversarial review → fix → verify, all in subagents. Eight must-fixes never reached live:

Would have happenedCaught in
Silent close-events — board drag-to-Done, supersede, duplicate emitted nothing; orchestrator never dispatches unblocked work#58 round 1
Session resurrection — steward nudge would respawn exited agents (~500MB each, stale context, acting unprompted)#59 round 1
Shell command injection — backticked nudge text (incl. agent-authored note content) typed into live bash = command substitution#59 round 1
Historical replay — first flag-flip would process weeks of dark-period events at once#59 round 1
Dedup collision — closing #5 silently swallowed by a prior "#55" comment; guaranteed past 10 issues#59 round 1
Keystroke injection — answer_question on a stale menu types digits (or a submitting Enter) into a working agent, reports success#62 round 1
Phantom phase events — one spurious row per session on every redeploy (restart re-seed)#58/#59
Vacuous idempotence test — the dedup test passed with the dedup deleted#59 round 1
Also positively verified each round: live-DB migration safety (all additive IF NOT EXISTS / guarded ALTER — zero boot risk), broadcast strip-safety for stale clients, hot-path cost, cursor monotonicity across pruning, and that every tool description matches its real server gate.

Enablement runbook (when you're ready)

  1. Optional pre-flight: land #63 (boot-seed mapper) so menus survive restarts as needs_user; without it, answer_question fails closed after a redeploy until the agent's next hook event.
  2. Flip: Settings → steward.enabled: true (takes effect within 15s, no restart).
  3. First tick seeds the cursor to "now" — dark history is skipped by design. Only new closes trigger.
  4. Watch: steward comments on issues ("Unblocked by #N: …"), podium issue events, and [podium:steward] journal lines. Every action is a comment — nothing is silent.
  5. Rollback: flip the flag off. The cursor persists; re-enabling resumes from where it stopped (or re-seeds if pruned past).
What enabling does NOT do: no auto-dispatch, no LLM calls, no merges, no session spawning. The only behaviors are unblock comments + nudges to live non-shell sessions. The pump (auto-dispatch) is P2 and unbuilt.

Deliberately not built yet

  • P2 — the pump: Interactive/Auto/Full autonomy on epics, budgets, caps, deterministic dispatch. Needs your design sign-off on the autonomy field UX + budget accounting unit (spec §8).
  • P3 — concierge: the + button intake flow, re-entry decision cards, artifact elicitation.
  • Steward LLM handlers: idle-session reconciliation and tidy tick (rule slots exist in TRIGGER_RULES; everything so far is deterministic — zero token cost).
  • Open small items: #63 boot-seed mapper; supersede is manage-gated (worker agents close-as-superseded instead — arguably correct).
