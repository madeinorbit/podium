01Colour is issue identity, never state. Every card tints with its issue's user-assigned palette colour (slate when unassigned) — the same colour the issue wears in the sidebar, tabs and workspace, so a glance maps card → issue. Here the failing card is teal and the merged one rose: state never picks the hue.
02State has its own reserved channels: amber attention chrome (ago-stamp, count pill, composer focus), working-green motion, and the mono state line. The issue palette deliberately excludes those hues, so identity and state can't collide.
03Arrival choreography (press the green demo button): the tray pill pops (popIn .45s), the card unfolds (.35s ease-out, grid-rows — no height snap), its surface flashes amber once (rowFlash .9s), the ago-stamp flips green→amber (flipAgo .5s), actions tick in (tickIn .35s). Then total stillness — stillness is the "needs you" signal. Reduced-motion: everything appears instantly.
04State line is machine-set from gitState: stage · branch · ahead-count · dirty/clean — exactly the fields the wire carries today. No invented test counts or ±line stats.
05Every action is a real affordance: buttons send their offer prompt to the agent session; Send back… collects feedback first (--action-input); session → opens the session. Nothing on the card pretends to be a view we don't have.
06The tray is global — every offer/question/finished item across all tasks, stable sort (decisions first, finished last, newest-first within each). The selected issue only adds the colour ring on POD-105. Changelog removed; chat owns the space.
07The super agent bar keeps its actions: open-in-terminal and clear context (sends /clear to the headless thread — one global chat, so clearing starts a fresh thread; the tray is untouched). Controls sit on the app's compact scale: bar icons 20px, card buttons 24px xs — same as the shipped UI, not smaller.
08The chat is column-sized: body 11px/1.5, labels 8.5px with mono timestamps, 8px rhythm. The super agent's own prompt now carries the answer-first response shape (lead with the answer, 1–3 sentences, numbered steps, restate in-flight state, ≤1 next action in mono amber, no preamble/closers) — the same discipline as the offer cards, tuned for a narrow column read at a glance.
