Agent chat view — Flat Field

The chosen direction for Podium's transcript view, built against the real message vocabulary the view renders today. The design manages attention the way the rest of the product does: the agent's work lies flat and quiet on the chassis, and only the surfaces that involve you — your prompts, a pending question, the agent's offer — get elevation or signal color.

What the view has to render (from chat.ts / ChatBlockView.tsx)

  • Operator prompts (+ file/image tags, sending / sent / failed)
  • Agent narration vs. the final Answer block
  • Tool batches — "Read 3 files, ran a command", expandable
  • Single Bash with the agent's own description
  • AskUserQuestion — interactive card, live-pending state
  • SendUserFile — image thumbnails → lightbox, file chips
  • Inter-agent mail envelopes (from → to, question / reply-requested, machine note)
  • System recap ("while you were gone"), churn dividers, interrupts
  • Machine-context rows (collapsed concierge seeds)
  • Search highlight/dim + minimap ticks

The design

Tuned for attention management. The agent's output lies flat on the chassis — no boxes, no rails, generous air — because it's the agent's room and its prose is the wallpaper. The one thing that pops is you: operator turns are the only elevated element, and they stick to the top while their turn scrolls (scroll the mock). Tool activity is visually muted but information-dense: every call type gets a purpose-built one-liner. Toggle the header switch to compare elevated vs. embossed user styling.

Why it works: attention is managed by elevation, not color — exactly one surface class (the operator) lifts off an otherwise flat field, so your eye finds the conversation's hinge points instantly, and the sticky prompt keeps "what did I ask for" pinned while you audit the work below it. Tool calls answer their own questions inline: Edits carry a proportional churn bar, a failed command shows its failing line without expanding, a subagent reports its one-line conclusion, and the live suite shows a ticking count. The Answer earns the page's only yellow — a keyline, not a box. An answered ask collapses to a one-line receipt (question + chosen option) so past decisions stay auditable without spending attention. The offer is designed in: it docks above the composer as the turn's single "needs you" surface — headline judged in five seconds, evidence thumbnails, primary action in yellow first, the feedback action (✎) swaps to an input field, a ghost for the low-commitment path. While an offer or a pending ask is live it owns the yellow; once consumed, the field goes flat again. The composer is the super-agent field verbatim: mono, > prompt, block caret, yellow focus border, mode hints below.

Design principles carried into implementation

  • Failure is currently invisible. Today a failed Bash call renders identically to a success until expanded. The design surfaces a per-call verdict; this is the single highest-leverage fix.
  • Answer blocks deserve the yellow. The Signal Rule says yellow marks what needs you — the final answer and a pending ask are exactly that; narration and tools must stay in the ink ramp.
  • Promote churn to the transcript. +118 −9 per tool stripe tells the operator the blast radius before they open anything.
  • Pending ask = stillness + timer. Per the motion grammar, the ask card shows a counting "waiting on you" timer; the spinner belongs to work, never to waiting.
  • Envelopes stay blue and framed — inter-agent mail must never be mistakable for the human, and the question badge is the operator's cue that another agent is blocked.
