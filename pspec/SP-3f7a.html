Right-click a sidebar session → Handoff → machine submenu moves a live session to another machine. Explicit user decisions (2026-07-14, #498; round-trip clarification 2026-07-18, POD-1013):

  • Move semantics — source agent is killed; the same session row re-homes to the target (no copy mode).
  • Worktree-only — only sessions in a worktree may hand off; the target always materializes a worktree and its main checkout is never touched.
  • Path-preserving round trips — preserve the source worktree location relative to the repository across machines. A handoff back to a machine that retained that checkout reclaims the same registered worktree in place; it must not silently switch between harness-owned .claude/worktrees/... and Podium-owned .worktrees/... paths.
  • Guarded residue reuse — leaving the source checkout is valid and preferred over destructive cleanup/recreation. Reclaim it only when it is the matching inactive handoff residue; never overwrite an unrelated, active, shared, or subsequently changed checkout merely because its branch matches.
  • Both harnesses in v1 — claude-code and codex.
  • Canonical session package — handoff rides a reusable package format (manifest.json + transcript + git bundle) intended later for export and harness-to-harness transfer, not ad-hoc plumbing.
  • Auto-clone deferred — v1 requires the repo on the target; cloning + GitHub credential propagation (#214) is a follow-up, done last.
