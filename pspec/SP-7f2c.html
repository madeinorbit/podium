Open-source release & distribution

Human decisions governing the public release of Podium (epic #12).

  • History: full filter-repo privacy rewrite (not squash-to-fresh-root); executed and force-pushed 2026-07-09.
  • License: Apache-2.0.
  • Library packages (@podium/protocol, agent-bridge, terminal-client): in-repo only at v1, no npm publish; private:true, versions stay 0.0.0.
  • Auth posture: keep warn-and-open when bound non-loopback without PODIUM_PASSWORD; every setup surface must make the no-password option an explicit opt-in with a confirmed warning. README documents this and recommends PODIUM_PASSWORD.
  • Distribution v1: GitHub Releases + SHA256SUMS. No self-hosted update feed (releases.podium.app deferred).
  • Desktop release cadence: desktop artifacts are never built for every push to main. Stable and edge desktop releases are explicitly cut/promoted; ordinary development does not use the production auto-updater path.
  • Desktop platform priority: macOS Apple Silicon is the primary desktop release target and explicit edge promotions publish its install and updater artifacts.
  • Update signing: production keypair generated by the human (private key never transits an agent session); interim throwaway dev keypair acceptable for end-to-end testing of podium update.
  • Windows: full experience desired incl. headless via a ConPTY (non-abduco) PTY backend (#90), built in a separate experimental worktree; does NOT block the OSS release.
  • Repo visibility: flipped public 2026-07-09 on user instruction, ahead of README/docs polish.
