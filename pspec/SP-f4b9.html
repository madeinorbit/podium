Pre-release gating for unfinished features (decided 2026-07-16). A feature-flag system hides incomplete work from users until it is ready.

Decisions

  • Every flag has a name and a description (user-facing copy, not just an id).
  • A settings page called Experimental lets users toggle experimental features on/off.
  • Each flag declares a visibility level controlling where it appears in Experimental:
    • hidden — not listed at all; enableable only via the config file — EXCEPT in development mode, where it is shown in Experimental.
    • by release channel — listed in Experimental for users on the matching update channel (stable/edge; edge-gated flags show only on edge installs).
  • Cmd+K search, the right-sidebar Git and Messages tabs, tab splitting, cross-server session handoff, workflows, specs, the Automations tab, and notifications are experimental features. Workflow and spec gates also hide their agent instructions and top-level CLI help entries.
