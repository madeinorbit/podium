An issue is never repurposed for something completely different: once named, it keeps its identity (decision 2026-07-16, Michael). When an agent starts on new work it must start a new issue or sub-issue rather than retitle/reuse the current one, and may attach itself to that new issue only on the human's push; otherwise it files the issue (or sub-issue) for another agent to implement. Surfaced to agents in issue prime (reads.ts), the claude-code system pointer, and docs/agents/podium-issues.md.
