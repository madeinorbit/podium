Shells exist in TWO places (revised 2026-07-20; supersedes the dock-only decision of 2026-07-13). (1) The right dock: the right rail has a Shell icon that opens a dock panel hosting one shell session per worktree, resolved from the active worktree. DOCK-owned shells never render as workspace agent tabs (the tab strip filters the dockShells map), and the worktree→session mapping persists so reloads reattach the same shell. (2) Workspace tabs: the workspace '+' (new tab) menu offers 'New Shell' alongside the agents; a shell spawned there is a regular tab session that participates in the panel system like a native agent. The sidebar New-work dropdown also offers 'New Shell' (spawns in the target repo's primary checkout as a draft). The sidebar per-row hover shell button (first #23 attempt) was explicitly rejected and reverted.

The shell panel identifies the machine on which its shell session is running (decided 2026-07-18).
