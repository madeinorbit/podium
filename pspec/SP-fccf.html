A Codex pane has a stable Podium session ID from creation. The native Codex thread ID may be absent until Codex starts a thread, including after the first submitted message.

When the native thread ID appears, Podium must attribute it to the exact pane that launched it, even when several new panes share a working directory or remain unsettled for different lengths of time. A sibling thread must never be inferred merely because it is the nearest or newest rollout.

Sidebar navigation uses the stable Podium session ID. Simultaneously live rows must remain independently visible and clickable if native identity data ever conflicts; identity deduplication must not silently redirect one live pane to another.
