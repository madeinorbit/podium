Human decisions (2026-07-20, POD-79/POD-82), building on the agent action offer channel [spec:SP-c7f1]:

  • The workspace Tray renders agent SessionOffers as cards with the agent's own dynamic action buttons — the same offer the chat composer bar and native PTY bar show. A button click sends its prompt to the offer's session as a normal user turn (server auto-clears the offer everywhere).
  • The old hardcoded review cards ("✓ Done — merge / Send back / Discuss") are REMOVED. An issue in review stage renders nothing in the Tray by itself; review-ready work announces itself via an offer. Question cards (needsHuman) remain.
  • Offers are stale-cleared server-side: a genuine entry into 'working' after the offer's createdAt (raw PTY input, mail/cron wakes, any continuation) consumes it — not only composer turns. At boot, persisted offers whose session shows later user input are dropped instead of replayed.
  • Clicking anywhere on a Tray card focuses the related native agent tab (offer cards open their exact session); inner action buttons stopPropagation.
  • The Tray also carries DETERMINISTIC cards recognized from issue state, not agent-offered: a human-audience task finished within the last 24h shows an Archive card (issues.archive = acknowledgment). The window is deliberately tighter than the sidebar's 7d unread visibility so historical never-read done issues cannot flood the tray.
