Agent sessions on remote machines forward browser-open intents through the owning daemon and server to attached clients. Clients confirm before opening the URL; the user click supplies the browser popup gesture. The server prefers clients focused on the session, then clients where it is visible, and otherwise notifies all connected clients. Requests remain available as needs-attention affordances when no client is connected.

While a forwarded login is pending, the client keeps a dismissible affordance that accepts the localhost callback URL reached by the user’s local browser. Podium sends the pasted URL back to the daemon that owns the session, which performs the HTTP GET against its own loopback listener. This user-initiated paste-back flow takes priority over a client-side loopback proxy and requires no client port binding.

The daemon binds callback execution to the pending open request, accepts only loopback HTTP URLs, matches the expected callback port and path derived from the original authorization URL when possible, and expires the capability when its affordance expires or is dismissed. Pasted callbacks are never executed against arbitrary hosts or ports.
